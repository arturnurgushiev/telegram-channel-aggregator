# Планы по развитию Telegram Channel Aggregator

## Поддержка медиафайлов
### Проблема
Насколько я понял Telegram Bot API не дает возможности просто взять и переслать (именно переслать) сообщение. Если бы это было возможно, то воспользовавшись методом forwardMessage мы бы возложили поддержку медиа на сам Telegram, но, к сожалению, этот метод работает только в случае, если бот добавлен в канал

### Возможные подходы
1. Создать буферный tg канал между userBot'ом и обычным tg ботом, тогда бот сможет пересылать сообщения прямо оттуда, сохраняя все медиафайлы

2. Реализовать всю поддержку media(загрузку, отправку) самостоятельно

3. Отказаться от telegram-bot-api b полностью перейти на gotd/td. Т.е взаимодействие с обычным тг ботом тоже переложить на эту библиотеку, возможно, в этом случае, за счет единообразия структур сообщений будет легче поддержать медиафайлы.


## Улучшение пользовательского опыта
- Команда `/list` - просмотр всех текущих подписок
- Команда `/help` - расширенная справка по использованию
- Команда `/stats` - базовая статистика по активности
- Автодополнение команд в интерфейсе бота
- Возможно добавление режима затухания для людей, которые считают себя зависимыми от всякого рода новостей и подписок. Бот будет присылать все меньше и меньше постов

## Категории и теги
- **Система категорий** - группировка каналов по темам
- **Автоматическая категоризация** - ML-классификация каналов

### Умная фильтрация контента
- **Умные фильтры** - фильтрация контента по категориям, количеству реакций, просмотров и.т.д
- **Ключевые слова** - фильтрация по заданным словам и фразам
- **Машинное обучение** - автоматическое определение релевантности, возможно хранить для каждого пользователя свою линейную или еще какую-нибудь модель

- **Персональные рекомендации** - умные предложения каналов. Пользователям с похожими моделями предлагать подписки друг друга
